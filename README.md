## What is Zyron?

Zyron is a production-grade Android AI assistant that replaces a single chatbot with a **coordinated swarm of four specialist agents** running in parallel. Every query is routed to a team of domain experts вҖ” an analyst, an executor, a validator, and a synthesizer вҖ” whose outputs are fused into a single, high-quality response.

Unlike standard AI apps where one model does everything, Zyron assigns each agent a strict role and a constitutional directive. They compete, cross-examine each other's work, and only the synthesized result reaches the user.

> **One question. Four agents. Zero compromise.**

The pipeline runs three phases:
1. **Analysis** вҖ” `queryAnalyzer.js` classifies intent, detects coding/STEM/creative signals, and selects COMPACT vs. FULL coordination mode
2. **Specialist execution** вҖ” Agents 1вҖ“3 run in parallel with individual SSE streaming, circuit-breakers, and automatic fallback chains
3. **Synthesis** вҖ” Agent 4 (Writer) receives a structured brief from Agents 1вҖ“3 and produces the final fused response, filtered by a quality judge and semantic deduplication

## Backend + Local Fallback Logic

Zyron uses a **dual-engine architecture** вҖ” a Railway-hosted Python FastAPI backend as the primary orchestrator, with a fully self-contained local JS engine as a silent fallback. The user never sees the switch happen.

### What the backend does
- **Python FastAPI** on Railway (`https://zyron-production-7af1.up.railway.app`)
- `POST /orchestrate` вҖ” accepts `{ query, agentConfigs, team, persona, userProfile }`
- Runs three specialist agents in parallel via `LangGraph` then synthesizes via the writer agent
- Returns structured `{ finalAnswer, agents[], coordinationMode }` JSON

### Fallback guarantee
- Timeout is **30 seconds**. If the backend doesn't respond in time, the local engine starts immediately вҖ” no error is ever shown to the user
- If the user pressed **Stop** while waiting, the abort propagates cleanly across both paths
- **Progress bar animation** starts on the frontend while the backend call is in-flight: agents animate from 0 % вҶ’ 78 % (exponential approach, П„ = 28 s), then jump to 100 % on response
- Agent names, icons, and accent colors are always **remapped to the active team** after a backend response вҖ” the backend drives logic, the frontend owns visual identity

### <svg width="18" height="18" viewBox="0 0 24 24" fill="none" xmlns="http://www.w3.org/2000/svg" style="vertical-align:middle"><rect x="9" y="2" width="6" height="11" rx="3" stroke="#7B2FFF" stroke-width="1.8"/><path d="M5 10a7 7 0 0014 0" stroke="#7B2FFF" stroke-width="1.8" stroke-linecap="round"/><path d="M12 21v-4" stroke="#7B2FFF" stroke-width="1.8" stroke-linecap="round"/><path d="M9 21h6" stroke="#7B2FFF" stroke-width="1.8" stroke-linecap="round"/></svg> Live Talk Mode *(New)*
Zyron can be talked to directly вҖ” no typing required. Live Talk opens a full-screen voice conversation interface powered by a neural-network node-web animation that pulses in sync with microphone activity.

**Pipeline:**
```
Mic Input (STT)  вҶ’  Transcript  вҶ’  LLM Call (Agent 1 config)  вҶ’  TTS Output
     вҶ‘                                                                в”Ӯ
     в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ Interrupt detection (new voice kills TTS) в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

- **Speech-to-text** via `expo-speech-recognition` вҖ” real-time partial results, silence detection
- **Continuous mic** вҖ” hardware never closes between turns (no click sound); a 1.5 s silence timer triggers the LLM call
- **Text-to-speech** via `expo-speech` вҖ” response is spoken sentence-by-sentence as tokens arrive, minimising latency
- **Interrupt** вҖ” if the user speaks while Zyron is talking, TTS stops immediately and the new input is processed
- **Auto-close** вҖ” if no speech is detected for 20 s after Zyron finishes speaking, the session closes silently
- **All providers supported** вҖ” uses Agent 1's configured key/model (OpenAI, Anthropic, Gemini, Groq, Mistral, DeepSeek, GLM, OpenRouter)
- **Conversation saved** вҖ” each turn is written to the active chat session in the background
- **Visual phases:** `idle` вҶ’ `listening` вҶ’ `thinking` вҶ’ `speaking` вҶ’ back to `listening`

## How to Use

### Step 1 вҖ” Add Your API Keys

1. Tap the **Settings** icon (top-right of the chat screen)
2. Navigate to **API Config**
3. For each agent socket (Agent 1 вҖ“ 4), select a provider from the dropdown and paste your API key
4. Tap **Verify** to confirm the key is live вҖ” a green status indicator means it's ready
5. Enable **Share key across agents** if you want all four sockets to use the same key

> You only need one working key to start. Zyron will use it across all agents until you configure individual keys.

---

### Step 2 вҖ” Choose a Team

1. In **Settings вҶ’ Agent Library**, browse the six specialist teams:
   `Financers В· Coders В· Scientists В· Mega Minds В· Historians В· Creative Thinkers`
2. Tap a team to preview its four agents and their roles
3. Tap **Set as Active Team** to activate it for your next query

The active team is shown in the header.

---

### Step 3 вҖ” Send a Query

1. Type your question in the input bar, or tap the **Mic** button to dictate
2. Tap **Send** (or press Enter)
3. Watch the four agent progress bars appear вҖ” each agent streams its output in real time
4. The final fused response appears in the chat bubble when Agent 4 completes

To **stop** a generation mid-stream, tap the **Stop** button.

---

### Step 4 вҖ” Use Live Talk

1. Tap the **Live** button in the input bar (right side)
2. The neural-network animation opens вҖ” tap the center to start listening
3. Speak your question вҖ” partial transcript shows in real time
4. After a 1.5 s pause Zyron thinks, then speaks the answer back to you
5. Keep talking for follow-up questions вҖ” the mic stays open continuously
6. Tap **Interrupt** to stop Zyron mid-sentence, or tap **X** to close the session

---

### Step 5 вҖ” Build a Custom Team

1. Go to **Settings вҶ’ Agents Workshop**
2. Switch to the **Agents** tab вҖ” create your custom agent personas with icons, traits, and strength sliders
3. Switch to the **Teams** tab вҖ” assign your agents to the four role slots
4. Give the team a name and icon, then tap **Create Team**
5. Your team now appears in Agent Library alongside the built-in teams

---

### Step 6 вҖ” Customize Synthesis Persona

1. Open **Settings вҶ’ Profile**
2. Under **Writer Persona**, pick one of five modes:

   | Persona | When to use |
   |---------|-------------|
   | **Balanced** | Everyday queries |
   | **Precise** | Technical / factual answers needing exact numbers |
   | **Educator** | Learning new concepts step-by-step |
   | **Creative** | Brainstorming, copywriting, storytelling |
   | **Executive** | Quick decisions вҖ” TL;DR + action item |

---

### Step 7 вҖ” Manage Conversations

| Action | How |
|--------|-----|
| **New chat** | Tap **пјӢ** in the top-left sidebar |
| **Switch session** | Slide open the left drawer and tap any past conversation |
| **Delete session** | Long-press a session in the sidebar вҶ’ Delete |
| **Search history** | Use the search bar at the top of the sidebar |

---

### Step 8 вҖ” Lock API Settings (Optional)

If other people use your device, enable **API Config Lock** in **Settings вҶ’ Privacy**:

1. Set a PIN or use biometric authentication
2. Once enabled, the **API Config** panel requires authentication before it can be opened
3. To remove the lock, authenticate and tap **Remove Lock**

---

### Tips

- **Offline?** Zyron falls back to on-device **Gemini Nano** automatically when all network providers are unreachable
- **Wrong team for your query?** The router will suggest a better-suited team вҖ” look for the suggestion banner below your response
- **Regenerate** a response by long-pressing the assistant bubble вҶ’ **Regenerate**
- **Copy code** from any code block by tapping the copy icon in the top-right of the block
- **Backend unavailable?** Zyron silently switches to its local engine вҖ” same quality, no interruption

## Getting Started

### Prerequisites

- Node.js 18+
- Expo CLI (`npm install -g expo-cli`)
- EAS CLI (`npm install -g eas-cli`) for builds
- At least one API key from any [supported provider](#supported-ai-providers)

### Installation

```bash
git clone https://github.com/NomanRafique01/zyron.git
cd zyron
npm install
```

### Running in Development

```bash
# Start Metro bundler
npx expo start

# Run on connected Android device / emulator
npx expo run:android
```

### Building a Release APK

```bash
# Preview APK (internal distribution)
eas build --profile preview --platform android

# Production APK
eas build --profile production-apk --platform android

# Production AAB (Play Store)
eas build --profile production --platform android
```

## Configuration

API keys are entered directly in the app's **Settings вҶ’ API Config** panel after launch. Keys are stored in Android Keystore-backed encrypted storage вҖ” never in source code or config files.

For `.env`-based secrets (used at build time only via `app.config.js`):

```bash
cp .env.example .env   # if provided
# or create manually вҖ” see app.config.js for expected variables
```

## Security

API keys live on the device. The app's security model is built around three principles:

1. **Encrypted storage** вҖ” Android Keystore-backed `EncryptedSharedPreferences` via `expo-secure-store`
2. **Single read path** вҖ” `keyGuard.js` is the only file that reads keys; no key is ever assigned to a module-level variable or logged
3. **Spend caps** вҖ” the most important protection is a hard monthly spend limit on every provider dashboard (see [`SECURITY.md`](SECURITY.md))

For production deployments requiring keys to leave the device entirely, `SECURITY.md` describes a ~50-line stateless Cloudflare Worker / Vercel Edge Function proxy that eliminates on-device key storage.
