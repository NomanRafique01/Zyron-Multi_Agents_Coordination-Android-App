## What is Zyron?

Zyron is a production-grade Android AI assistant that replaces a single chatbot with a **coordinated swarm of four specialist agents** running in parallel. Every query is routed to a team of domain experts ‚ÄĒ an analyst, an executor, a validator, and a synthesizer ‚ÄĒ whose outputs are fused into a single, high-quality response.

Unlike standard AI apps where one model does everything, Zyron assigns each agent a strict role and a constitutional directive. They compete, cross-examine each other's work, and only the synthesized result reaches the user.

> **One question. Four agents. Zero compromise.**

The pipeline runs three phases:
1. **Analysis** ‚ÄĒ `queryAnalyzer.js` classifies intent, detects coding/STEM/creative signals, and selects COMPACT vs. FULL coordination mode
2. **Specialist execution** ‚ÄĒ Agents 1‚Äď3 run in parallel with individual SSE streaming, circuit-breakers, and automatic fallback chains
3. **Synthesis** ‚ÄĒ Agent 4 (Writer) receives a structured brief from Agents 1‚Äď3 and produces the final fused response, filtered by a quality judge and semantic deduplication

## Feature Overview

### ūü§Ė Multi-Agent Swarm Engine
- **Parallel execution** ‚ÄĒ Agents 1‚Äď3 fire simultaneously, not sequentially
- **Team Blending** ‚ÄĒ cross-team specialist borrowing per query without changing the active session team (e.g., pull Mega Minds' Scholar into a Coders session)
- **Coordination modes** ‚ÄĒ `NONE` (direct), `COMPACT` (brief sharing), `FULL` (structured specialist briefs with shared context)
- **Model tier routing** ‚ÄĒ cheap vs. expensive model selection based on query complexity
- **Team switching suggestions** ‚ÄĒ the router detects when a different team would produce better results

### ‚ö° Real-Time Streaming
- **Server-Sent Events (SSE)** for all supporting providers ‚ÄĒ tokens stream live to screen
- **Per-agent streaming state** ‚ÄĒ each agent has its own progress bar and status label (Thinking / Building / Red-teaming / Polishing)
- **Graceful abort** ‚ÄĒ stop mid-generation, generation cancels cleanly across all active sockets

### ūüõ°ÔłŹ Resilient API Layer
- **Circuit breaker** ‚ÄĒ per-provider failure tracking; tripped circuits skip that socket for the current session
- **Fallback chains** ‚ÄĒ automatic failover to the next provider on timeout or error
- **Sanitized error messages** ‚ÄĒ no raw API keys or stack traces ever surface in the UI

### ūüíĺ On-Device Memory
- **SQLite persistence** ‚ÄĒ full conversation history with session index and message pagination
- **User memory** ‚ÄĒ contextual facts extracted from conversations and injected into future prompts
- **Offline fallback** ‚ÄĒ Gemini Nano on-device inference for queries when all network providers are unreachable

### ūüĒí Security
- **Android Keystore-backed storage** ‚ÄĒ API keys stored in `EncryptedSharedPreferences` via `expo-secure-store`; never in `AsyncStorage` or any JS bundle string
- **Single read gateway** ‚ÄĒ `keyGuard.js` is the only permitted key-read path; keys are read at call-time and never stored in module-level variables
- **Optional API Config Lock** ‚ÄĒ a password gate that locks all API settings behind biometric or PIN authentication
- **No telemetry** ‚ÄĒ all telemetry is session-local, never transmitted

### ‚úćÔłŹ Synthesis Intelligence
- **Quality judge** ‚ÄĒ LLM + heuristic scorer evaluates output against domain-specific required/prohibited criteria (coding, STEM, analytical, writing, creative, general)
- **Semantic deduplication** ‚ÄĒ removes redundant content across agent outputs before synthesis
- **Agent personas** ‚ÄĒ five Writer synthesis modes: Balanced, Creative, Precise, Educator, Executive

### ūüßģ Math & Code Rendering
- **LaTeX rendering** ‚ÄĒ `KaTeX` via `WebView` for inline and display math formulas
- **Syntax-highlighted code blocks** ‚ÄĒ dedicated `SyntaxCode` component with language detection and copy-to-clipboard
- **Chemical formula detection** ‚ÄĒ `mathParser.utils.js` handles inline chemical notation alongside LaTeX

### ūüóāÔłŹ Conversation Management
- **Timeline grouping** ‚ÄĒ conversations bucketed into Today / Yesterday / Older
- **Session sidebar** ‚ÄĒ slide-out drawer with conversation history, search, and new-chat creation
- **Smart welcome greeting** ‚ÄĒ time-aware greeting (Night Owl / Early Bird / Golden Hour / etc.) with the user's first name

### ‚öôÔłŹ Settings & Personalization
- **Profile panel** ‚ÄĒ display name, role, preferred tone, language, coding style, detail level
- **Agent Library** ‚ÄĒ accordion panel showing all six teams with per-team agent roster and expand-to-inspect detail
- **API Config panel** ‚ÄĒ per-provider key entry, model selection, key status verification, share-key-across-agents toggle
- **Privacy panel** ‚ÄĒ privacy mode, profile context injection toggle
- **Reset panel** ‚ÄĒ wipe conversation history, clear API keys, full factory reset

## Getting Started

### Prerequisites

- Node.js 18+
- Expo CLI (`npm install -g expo-cli`)
- EAS CLI (`npm install -g eas-cli`) for builds
- At least one API key from any [supported provider](#supported-ai-providers)

### Installation

```bash
git clone https://github.com/NomanRafique01/zyron.git
cd zyron
npm install
```

### Running in Development

```bash
# Start Metro bundler
npx expo start

# Run on connected Android device / emulator
npx expo run:android
```

### Building a Release APK

```bash
# Preview APK (internal distribution)
eas build --profile preview --platform android

# Production APK
eas build --profile production-apk --platform android

# Production AAB (Play Store)
eas build --profile production --platform android
```

API keys are entered directly in the app's **Settings ‚Üí API Config** panel after launch. Keys are stored in Android Keystore-backed encrypted storage ‚ÄĒ never in source code or config files.

## Security

API keys live on the device. The app's security model is built around three principles:

1. **Encrypted storage** ‚ÄĒ Android Keystore-backed `EncryptedSharedPreferences` via `expo-secure-store`
2. **Single read path** ‚ÄĒ `keyGuard.js` is the only file that reads keys; no key is ever assigned to a module-level variable or logged
3. **Spend caps** ‚ÄĒ the most important protection is a hard monthly spend limit on every provider dashboard (see [`SECURITY.md`](SECURITY.md))

For production deployments requiring keys to leave the device entirely, `SECURITY.md` describes a ~50-line stateless Cloudflare Worker / Vercel Edge Function proxy that eliminates on-device key storage.
