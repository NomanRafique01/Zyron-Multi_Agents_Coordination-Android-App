## What is Zyron?

Zyron is a production-grade Android AI assistant that replaces a single chatbot with a **coordinated swarm of four specialist agents** running in parallel. Every query is routed to a team of domain experts вҖ” an analyst, an executor, a validator, and a synthesizer вҖ” whose outputs are fused into a single, high-quality response.

Unlike standard AI apps where one model does everything, Zyron assigns each agent a strict role and a constitutional directive. They compete, cross-examine each other's work, and only the synthesized result reaches the user.

> **One question. Four agents. Zero compromise.**

The pipeline runs three phases:
1. **Analysis** вҖ” `queryAnalyzer.js` classifies intent, detects coding/STEM/creative signals, and selects COMPACT vs. FULL coordination mode
2. **Specialist execution** вҖ” Agents 1вҖ“3 run in parallel with individual SSE streaming, circuit-breakers, and automatic fallback chains
3. **Synthesis** вҖ” Agent 4 (Writer) receives a structured brief from Agents 1вҖ“3 and produces the final fused response, filtered by a quality judge and semantic deduplication

## Feature Overview

### рҹӨ– Multi-Agent Swarm Engine
- **Parallel execution** вҖ” Agents 1вҖ“3 fire simultaneously, not sequentially
- **Team Blending** вҖ” cross-team specialist borrowing per query without changing the active session team (e.g., pull Mega Minds' Scholar into a Coders session)
- **Coordination modes** вҖ” `NONE` (direct), `COMPACT` (brief sharing), `FULL` (structured specialist briefs with shared context)
- **Model tier routing** вҖ” cheap vs. expensive model selection based on query complexity
- **Team switching suggestions** вҖ” the router detects when a different team would produce better results

### вҡЎ Real-Time Streaming
- **Server-Sent Events (SSE)** for all supporting providers вҖ” tokens stream live to screen
- **Per-agent streaming state** вҖ” each agent has its own progress bar and status label (Thinking / Building / Red-teaming / Polishing)
- **Graceful abort** вҖ” stop mid-generation, generation cancels cleanly across all active sockets

### рҹӣЎпёҸ Resilient API Layer
- **Circuit breaker** вҖ” per-provider failure tracking; tripped circuits skip that socket for the current session
- **Fallback chains** вҖ” automatic failover to the next provider on timeout or error
- **Sanitized error messages** вҖ” no raw API keys or stack traces ever surface in the UI

### рҹ’ҫ On-Device Memory
- **SQLite persistence** вҖ” full conversation history with session index and message pagination
- **User memory** вҖ” contextual facts extracted from conversations and injected into future prompts
- **Offline fallback** вҖ” Gemini Nano on-device inference for queries when all network providers are unreachable

### рҹ”’ Security
- **Android Keystore-backed storage** вҖ” API keys stored in `EncryptedSharedPreferences` via `expo-secure-store`; never in `AsyncStorage` or any JS bundle string
- **Single read gateway** вҖ” `keyGuard.js` is the only permitted key-read path; keys are read at call-time and never stored in module-level variables
- **Optional API Config Lock** вҖ” a password gate that locks all API settings behind biometric or PIN authentication
- **No telemetry** вҖ” all telemetry is session-local, never transmitted

### вңҚпёҸ Synthesis Intelligence
- **Quality judge** вҖ” LLM + heuristic scorer evaluates output against domain-specific required/prohibited criteria (coding, STEM, analytical, writing, creative, general)
- **Semantic deduplication** вҖ” removes redundant content across agent outputs before synthesis
- **Agent personas** вҖ” five Writer synthesis modes: Balanced, Creative, Precise, Educator, Executive

### рҹ§® Math & Code Rendering
- **LaTeX rendering** вҖ” `KaTeX` via `WebView` for inline and display math formulas
- **Syntax-highlighted code blocks** вҖ” dedicated `SyntaxCode` component with language detection and copy-to-clipboard
- **Chemical formula detection** вҖ” `mathParser.utils.js` handles inline chemical notation alongside LaTeX

### рҹ—ӮпёҸ Conversation Management
- **Timeline grouping** вҖ” conversations bucketed into Today / Yesterday / Older
- **Session sidebar** вҖ” slide-out drawer with conversation history, search, and new-chat creation
- **Smart welcome greeting** вҖ” time-aware greeting (Night Owl / Early Bird / Golden Hour / etc.) with the user's first name

### вҡҷпёҸ Settings & Personalization
- **Profile panel** вҖ” display name, role, preferred tone, language, coding style, detail level
- **Agent Library** вҖ” accordion panel showing all six teams with per-team agent roster and expand-to-inspect detail
- **API Config panel** вҖ” per-provider key entry, model selection, key status verification, share-key-across-agents toggle
- **Privacy panel** вҖ” privacy mode, profile context injection toggle
- **Reset panel** вҖ” wipe conversation history, clear API keys, full factory reset

## Security

API keys live on the device. The app's security model is built around three principles:

1. **Encrypted storage** вҖ” Android Keystore-backed `EncryptedSharedPreferences` via `expo-secure-store`
2. **Single read path** вҖ” `keyGuard.js` is the only file that reads keys; no key is ever assigned to a module-level variable or logged
3. **Spend caps** вҖ” the most important protection is a hard monthly spend limit on every provider dashboard (see [`SECURITY.md`](SECURITY.md))

For production deployments requiring keys to leave the device entirely, `SECURITY.md` describes a ~50-line stateless Cloudflare Worker / Vercel Edge Function proxy that eliminates on-device key storage.
